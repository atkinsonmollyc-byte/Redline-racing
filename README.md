# Redline Racing Website

## What is This?

This is a **fictional motorsports team website** built with HTML and CSS. It's a learning project designed to practice front-end web development skills—no real racing team or people involved.

## About Redline Racing

Redline Racing is a **completely fictional** professional motorsports team. The drivers, race results, and team details are all made up for this project. This is a practice website, not a real organization.

## Pages & Features

The website includes:

- **Homepage** (`index.html`)
  - Hero section with team branding
  - Intro to the team
  - Featured drivers section
  - Race car profile
  - Team statistics and timeline
  - Next race countdown timer
  - Navigation bar

- **Driver Profile Pages**
  - Alex Hart (`alex-hart.html`)
  - Luca Rossi (`luca-rossi.html`)
  - James Holleway (`james-holleway.html`)
  - Hazel Mitehall (`hazel-mitehall.html`)
  - Each with bio, achievements, and stats

- **2025 Season Hub** (`season-2025.html`)
  - Race schedule and results table
  - Season storyline and objectives
  - Team standings

- **News Centre** (`news.html`)
  - Fictional news articles about the team
  - News cards with publication dates and tags

- **Navigation**
  - Consistent site-wide navigation bar
  - Back links on all pages
  - Theme-aligned link styling

## How to Open It Locally

1. Download or save all the files to a folder on your computer
2. Open `index.html` in any web browser (Chrome, Firefox, Safari, Edge, etc.)
   - On Windows: Right-click `index.html` → "Open with" → choose your browser
   - On Mac: Double-click `index.html` to open in your default browser
3. Use the navigation bar to explore all pages

**No server or special setup needed—it's just HTML and CSS!**

## What I Learned

Building this website taught me:

- **HTML Structure** – Creating semantic page layouts with sections, headers, and navigation
- **CSS Styling** – Using Flexbox and CSS Grid for responsive layouts
- **Responsive Design** – Using `clamp()` functions and media queries to make the site work on different screen sizes
- **Design Consistency** – Keeping colors, fonts, and spacing consistent across multiple pages
- **User Experience** – Adding focus states, hover effects, and countdown timers for better interactivity
- **Branding** – Using a cohesive color scheme (dark background, red accents, gold highlights) throughout the site

## Safety & Guardrails

**No Private Information:**
- All team names, driver names, and biographical details are fictional
- No real addresses, phone numbers, or personal data included
- No external APIs or databases that could expose data
- Self-contained HTML and CSS files (no external dependencies)

**What This Means:**
- The website is safe to share and edit
- No privacy concerns—everything is made up
- Great for learning without worrying about protecting real information
